Hexo博客搭建

发布于 2024-03-01 折腾 

前言

在我搭建博客之前,我调研了市面上大家常用的博客框架,其中最著名,使用人数最多当属WordPress,但是在我实际使用之后,我发现我并不需要这么重的一个博客框架,WordPress其中的大部分功能我都用不到.在这过程中,我也渐渐清晰了我的需求,我所需要的是一个简单,简洁的博客框架,且可以使用markdown来编写博客,这时我发现了Hexo.

其实Hexo的简单,纯静态只是我选择它的一半的原因,另一个原因是Hexo的一个主题,也就是我现在当前博客使用的主题typography,这个主题实在是太符合我的审美了,虽然作者现在好像也不维护了,但我依然要大力推荐!(自己稍微改改代码也不是不行嘛)

安装

作为一个轻量级的博客框架,安装也是非常简单的(官方文档)

1.安装git

2.安装node.js

3.安装Hexo

# 安装 hexo
hexo npm install hexo-cli g 
# 初始化博客文件
hexo init blog
# 切换到该路径
cd blog
# 安装hexo的扩展插件
npm install

4.安装主题

npm uninstall --save hexo-generator-category 
npm install --save hexo-renderer-pug hexo-generator-archive hexo-generator-category-enhance hexo-generator-feed hexo-generator-tag
npm install --save hexo-prism-plugin 

git clone https://github.com/orrrpuse/hexo-theme-typography.git themes/typography
cd themes/typography
npm install

配置文件

首先是根目录下Hexo的配置文件(官方文档)

其次是/theme/typography目录下主题的配置文件(主题README配置文档)

运行

注意,要在Hexo根目录下执行命令

hexo clean && hexo g && hexo s

写作

可以使用官方命令

hexo new <标题名>

或者直接复制你的markdown文件到/source/_posts 中

最后

因为是静态文件,想要部署博客可以用nginx直接引用.

或者官方也提供了开启web服务的命令

hexo s

不过我更推荐另一种更加优雅的方式:

通过官方提供的一键部署的方式直接部署到GitHub Pages,可以实现提交新的文章之后自动部署到GitHub Pages上
